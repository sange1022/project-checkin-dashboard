# 每日随机内容设计

日期：2026-06-21

## 目标

在时间视图切换器下方增加三个极简的每日随机内容入口：健身、建模、建脑。每个分类每天只能抽取一次，结果保存在浏览器本地。

## 默认候选

- 健身：俯卧撑、深蹲、走路
- 建模：艺术史、建筑史、访谈、古典乐、AI、美食
- 建脑：历史、哲学、人性、传记、物理、文学

## 主界面

- 三个分类位于“每日 / 每周 / 每月”切换器正下方，横向三列。
- 未抽取时显示“点击随机”。
- 点击后候选文字快速轮换 2 秒。
- 2 秒后停在最终结果并锁定。
- 同一分类当天不能再次抽取。
- 刷新页面保持当天结果，次日自动恢复可抽取。
- 三列在所有时间视图中均显示。

## 本地数据

- `randomCategories` 保存三个分类及候选项，每个候选项有稳定 ID 和可编辑名称。
- `dailyRandomResults` 按本地 ISO 日期保存三个分类当天最终结果。
- 结果同时保存候选项 ID 和抽取时名称，后续修改候选名称不改写历史。
- 数据纳入现有本地存储及 JSON 导入导出。

## 管理随机内容

- 页面最下方增加折叠区“管理随机内容”。
- 按分类显示候选列表。
- 候选名称可原地编辑。
- 每类可添加和删除候选项。
- 每类至少保留一个非空候选项。
- 分类名称固定为健身、建模、建脑，本期不支持新增或删除分类。

## 随机记录

- 页面最下方增加折叠区“随机记录”。
- 按日期倒序展示当日最终结果。
- 每个日期只展示已有结果；当天未抽取的分类留空。
- 历史只读，不提供删除或修改。

## 动画和交互

- 动画持续约 2 秒，每 80–120 毫秒轮换一次候选文字。
- 动画期间三个分类可分别并行抽取。
- 动画期间当前分类按钮禁用。
- 最终结果由点击时随机选择并在动画结束时保存。
- 页面遵循 `prefers-reduced-motion`：减少动态时只显示短暂等待，不快速轮换。

## 验收

- 每类每天只能成功抽取一次。
- 三类结果均能独立保存并在刷新后恢复。
- 次日可重新抽取。
- 候选内容可以添加、编辑和删除。
- 空候选名称不保存，最后一项不可删除。
- 随机记录按日期倒序显示。
- 旧版本地数据可无损升级，现有项目和打卡记录不丢失。
