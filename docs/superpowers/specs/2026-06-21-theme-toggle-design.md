# 日夜主题切换设计

日期：2026-06-21

## 目标

在页面右上角增加极简日夜主题切换，并保证所有现有界面在夜晚模式下保持清晰、克制和一致。

## 交互

- 切换按钮位于搜索按钮左侧，与现有图标按钮同尺寸。
- 白天模式显示月亮图标，表示可切换到夜晚。
- 夜晚模式显示太阳图标，表示可切换到白天。
- 首次打开且未保存选择时，跟随操作系统 `prefers-color-scheme`。
- 用户手动切换后保存为 `light` 或 `dark`，后续不再跟随系统变化。
- 按钮提供明确的无障碍名称。

## 视觉

- 夜晚模式不用纯黑，页面外层为深灰，主画布为略浅深灰。
- 正文使用柔和白色，次要文字使用中灰。
- 边框、未打卡格、输入框、折叠区、随机区和项目阶段轴全部使用统一深色变量。
- 已打卡格与进度线在夜间改为浅色，以保持对比。
- 行列悬停使用略亮深灰，不覆盖已打卡状态。
- 颜色切换约 150ms；`prefers-reduced-motion` 下取消过渡。

## 数据

- `theme` 保存 `light | dark | system`。
- 新用户默认为 `system`。
- 手动点击后写为 `light` 或 `dark`。
- 主题字段纳入浏览器本地存储和 JSON 导入导出。
- 旧版本地数据自动补 `system`。

## 验收

- 首次打开跟随系统主题。
- 手动切换后刷新仍保持。
- 全部页面区域在两种主题下可读。
- 图标与当前可执行动作一致。
- JSON 导入导出保留主题选择。
