# 项目阶段进度视图设计

日期：2026-06-21

## 目标

在每日打卡项目表格正下方直接展示“项目进度视图”，用横向阶段轴展示独立项目当前所处的设计服务阶段和自动百分比。

## 项目

- 该区域独立维护项目，不与上方每日打卡项目同步。
- 最下方提供折叠式“管理进度项目”，用于新增、原地修改和删除。
- 新增完成后管理区自动收起。
- 上方进度视图只负责查看项目名称、点击阶段和查看百分比。
- 新建项目默认处于“第一次沟通”，进度为 10%。

## 固定阶段

1. 第一次沟通 — 10%
2. 交付设计定金 — 20%
3. 现场量尺 — 30%
4. 平面方案设计 — 40%
5. 第一次方案沟通 — 50%
6. SU建模方案设计中 — 60%
7. 第二次方案沟通 — 65%
8. 效果图制作中 — 70%
9. 效果图沟通 — 75%
10. 施工图纸制作中 — 80%
11. 对接施工图纸 — 85%
12. 软装搭配PPT制作中 — 90%
13. 软装PPT交付 — 95%
14. 现场施工中 — 96%
15. 设计跟踪服务完结 — 100%

阶段名称与百分比固定，不提供编辑。

## 交互和视觉

- 每个项目占一行，阶段沿横向排列。
- 点击某个阶段节点立即将该项目更新到该阶段。
- 一条较粗深色线从起点延伸到当前阶段。
- 已经过的节点加深，未到达节点保持浅灰。
- 行末固定显示当前百分比。
- 阶段名称较长时区域横向滚动，项目名称固定在左侧。
- 使用现有宋体和黑灰极简视觉，不增加彩色状态。
- 进度视图位于每日打卡表格之后、空状态之前。

## 数据

- `stageProjects` 独立保存在 `AppState`。
- 每项包含 ID、名称、当前阶段索引和创建时间。
- 纳入浏览器本地存储和 JSON 导入导出。
- 旧版本地数据自动补为空数组，不影响现有项目、打卡和随机记录。

## 验收

- 能新增、改名、删除阶段项目。
- 新项目默认 10%。
- 点击 15 个阶段中的任意节点后，线、节点和百分比同步变化。
- 刷新后阶段项目和进度保持。
- JSON 导入导出保留阶段项目。
- 窄屏可横向滚动且项目名称固定。
